# MemoMoss

## Problem Statement

Alzheimer's and dementia patients often struggle to remember important people, medications, schedules, and daily activities. Caregivers also face difficulties in monitoring and assisting patients while ensuring their safety and independence.

## Proposed Solution

MemoMoss is a caregiver-assisted memory companion platform designed to support people with dementia and Alzheimer's disease. The platform helps patients remember important events, manage daily routines, access emergency assistance, and stay connected with caregivers.

## Key Features

### Patient Dashboard

- Chat-based assistance
- Voice query support
- Face recognition for familiar people
- Chronological memory timeline
- SOS emergency system
- Medication and schedule reminders

### Caregiver Dashboard

- Manage reminders and schedules
- Upload memories and important events
- Request authorization
- Monitor patient wellbeing
- Manage emergency contacts

### Cognitive Support

- Brain Gym activities
- Memory exercises
- Language and word processing tests
- Basic mathematics exercises
- Progress tracking

### Safety Features

- Patient verification through voice and face authentication
- Guardian authorization workflow
- Multiple SOS contacts
- Future geofencing support

## Technology Stack

- Frontend: HTML, CSS, JavaScript
- Backend: Node.js, Express.js
- Database: MongoDB Atlas
- Version Control: GitHub

## Project Status

This project is currently under active development. The repository contains the backend foundation, database design, React frontend dashboards, Brain Gym prototype, caregiver interface, and overall system architecture for MemoMoss. Frontend and backend modules are currently being developed independently and integration is in progress.


## Current Status

- Backend foundation implemented using Node.js, Express, MongoDB Atlas, and Mongoose.
- Patient registration API implemented and tested.
- React frontend developed with separate Login, Patient Dashboard, Caregiver Dashboard, Brain Gym, and Memory Timeline pages.
- Navigation between frontend pages implemented using React Router.
- Brain Gym and dashboard interfaces currently function as frontend prototypes.
- Backend and frontend are currently developed separately and are not yet connected through API integration.
- User authentication, dashboard data fetching, reminders, memory records, and Brain Gym progress are not yet connected to the backend database.
- Ongoing work includes frontend-backend integration, real-time data handling, and user-specific functionality.
